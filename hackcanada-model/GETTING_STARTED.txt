🎓 THREE CLASS SYSTEM

Label your images with exactly one of these:

CLEAN (100% refund)
├─ Zero visible damage
├─ Pristine condition
└─ Examples: Product photos, new listings

LIGHT_DAMAGE (65% refund)
├─ Single or paired minor issues
├─ Localized damage
├─ Examples: Small stain, light scratches, minor tear
└─ Still returnable!

HEAVY_DAMAGE (0% refund / reject)
├─ Multiple issues OR severe single damage
├─ Affects usability/structure
├─ Examples: Large rips, broken frame, heavy stains
└─ Return rejected

⚙️ TRAINING PIPELINE (What train.py Does)

1. Loads images from data/raw/CLEAN/, LIGHT_DAMAGE/, HEAVY_DAMAGE/
2. Creates splits:
   - 70% training (210 images)
   - 15% validation (45 images)
   - 15% test (45 images)
3. Applies data augmentation to training set
4. Fine-tunes MobileNetV2:
   - Frozen early layers (transfer learning)
   - New 3-class classifier
   - Adam optimizer + learning rate scheduling
5. Saves best model to: models/best_model_TIMESTAMP.pth
6. Shows epoch-by-epoch accuracy/loss

Expected training time:
- CPU: 45-90 minutes
- GPU: 10-15 minutes
- Expected accuracy: 75-85% on validation set
